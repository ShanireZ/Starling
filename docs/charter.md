# Starling v2 立项文档

> **一句话**:装在 GSX250R **车身侧板(车把下整流罩)**上的一对**电子百叶进气口("鳃口")**(固定框 + 联动叶片),**开口朝车头迎风**,随车速开合——越快开得越大,像**张鳃呼吸**纳入迎面气流。**看得见的动态造型件,不是性能件。**

立项日期:2026-06-25 · 名称沿用 Starling(v1 同名项目已废弃重启)

---

## 0. 为什么重启:v1 的教训

v1(摩托车主动空气动力学系统)流产,四个失败点其实是同一个病根的四张脸——**成本前置、收益后置,而收益天花板本来就低**:

| v1 失败点 | 实质 |
|---|---|
| 设计过于复杂,逐渐流产 | 成本无限后拖,从未产出可用切片 |
| 成本 ≫ 收益(仅很小提升) | **真正死因**:250 摩托 <100km/h,主动气动的下压力收益微乎其微 |
| 结构复杂、不便调试观察 | 烧钱过程中看不见盈亏 |
| 过于专业 | 锁死在一个人脑子里,无人能验 |

**复杂本身不是罪,复杂配不上收益才是。** v2 由此立下两道闸:

1. **立项前给收益封顶** —— 既然性能收益注定小,就不做性能件,改做*视觉 / 动态件*,把成本对齐到"炫酷"这个可达又可观察的价值上。
2. **第一周就要有看得见的纵切片** —— 用最小可观察实物验证,而非建几个月才发现不值。

## 1. 项目身份

- **名称**:Starling(沿用并重释义——椋鸟 / 凤凰,展翅母题)
- **平台**:Suzuki **GSX250R**(单车型起步,非通用支架)
- **形态(2026-06-25 大改 → 百叶式;2026-06-28 定位侧板 + 开口朝车头迎风)**:**车身侧板(车把下整流罩)**上一对**电子百叶进气口("鳃口")**——一个**固定的 U / 矩形结构框**(内凹、NACA 式观感),框内一组**多片联动百叶**,**开口朝车头迎风**。低速齐平闭合,高速叶片张开露缝(战斗机 / F1 多段开缝层叠面的样子)。
- **行为**:**速度越快 → 开度越大**(MCU 按速度驱动百叶角度);开口朝前,张开时像「张鳃呼吸」纳入迎面气流。
- **机构**:固定框承受风载;百叶 = **一个舵机 + 一根连杆带动多片 = 1 个真实自由度**(汽车主动进气格栅 / 空调出风口的成熟做法)。
- **气动姿态(只求无害,不求收益)**:① **开口朝车头迎风**,低速齐平闭合(最低阻),高速张开像「张鳃呼吸」纳入迎面气流;② 固定框包住叶片边缘(少抖振 / 哨音);③ **不做真涵道 / 进气散热**——只借迎风口的观感,不做真穿流;高速张开会有少量增阻(迎风口纳气),在 <100km/h、小开口下视为**无害可接受**;不追求下压力 / 散热 / 性能。
- **无灯光**:视觉价值 = 这套随速度开合的**动态** + 造型本身。

---

## 2. 定位与价值(收益封顶)

- **价值锚点**:视觉 / 动态炫酷("展翅感"),外行一眼能看懂
- **明确不为性能买单**:不追求下压力、不做气动调优、不测性能增益
- **场景**:100km/h 以内的日常骑行
- **ROI 闸门**:任何"为多一点性能 / 下压力"的复杂度,默认砍掉

---

## 3. 控制架构(防战线蔓延的生死线)

- **实时控制 100% 板载自包含**:MCU + 传感器,本地决定翼面开度
- **外部连接(蓝牙或有线)只做两件事**:① 配置(模式 / 参数) ② 数据采集与回传
- **外部端(手机 / PC / 网页)不参与实时控制**
- **无云、无 OTA、无跨学科集成**

---

## 4. 设计铁律(v1 四大死因的对策)

1. **收益封顶** — 价值是视觉,不是性能;砍掉一切性能向复杂度。
2. **第一周可观察纵切片** — 先做出"翼面随某个输入张合"的能看见的最小原型。
3. **单人可造、可调、可调试** — 不引入需专家 / 多人才能维护的结构。
4. **禁战线蔓延** — 无 App 实时控制、无云、无 OTA、无应变片性能反馈链、无多学科集成计划。
5. **外行可读** — 不懂的人看一眼就明白"它随速度展翅,很酷"。

---

## 5. 范围

### ✅ 做(In)
- **机械**:固定 U / 矩形框(兼车头安装支架)+ 多片联动百叶 + 单舵机连杆
- **电子**:板载 MCU + 测速传感 + 供电 + 舵机驱动
- **软件**:板载实时控制逻辑(速度 → 开度)+ 极简配置 / 数据接口(蓝牙或有线)

### ❌ 不做(Out — 明确放弃,守住边界)
- 主动下压力优化 / 闭环气动调参 / 借襟翼追升力
- **真功能性涵道 / 进气散热**(只借观感,不做真穿流)
- 配套 App 实时控制 / 云遥测 / OTA
- 应变片性能反馈链 / 五学科集成计划(v1 的坟墓)
- 灯光
