# Starling v2 立项文档

> **一句话**:装在摩托车车头两侧的 C 形**电子凤翼**,随车速张合——速度越快、正面气流越强,翼面开得越大,像凤凰展翅。**看得见的动态造型件,不是性能件。**

立项日期:2026-06-25 · 名称沿用 Starling(v1 同名项目已废弃重启)

---

## 0. 为什么重启:v1 的教训

v1(摩托车主动空气动力学系统)流产,四个失败点其实是同一个病根的四张脸——**成本前置、收益后置,而收益天花板本来就低**:

| v1 失败点 | 实质 |
|---|---|
| 设计过于复杂,逐渐流产 | 成本无限后拖,从未产出可用切片 |
| 成本 ≫ 收益(仅很小提升) | **真正死因**:250 摩托 <100km/h,主动气动的下压力收益微乎其微 |
| 结构复杂、不便调试观察 | 烧钱过程中看不见盈亏 |
| 过于专业 | 锁死在一个人脑子里,无人能验 |

**复杂本身不是罪,复杂配不上收益才是。** v2 由此立下两道闸:

1. **立项前给收益封顶** —— 既然性能收益注定小,就不做性能件,改做*视觉 / 动态件*,把成本对齐到"炫酷"这个可达又可观察的价值上。
2. **第一周就要有看得见的纵切片** —— 用最小可观察实物验证,而非建几个月才发现不值。

---

## 1. 项目身份

- **名称**:Starling(沿用并重释义——椋鸟 / 凤凰,展翅母题)
- **形态**:摩托车**车头两侧、C 形可动翼面**(俗称"凤翼")
- **行为**:开度随**正面气流大小 + MCU 加减速状态**变化;**速度越快 → 气流越强 → 开度越大**
- **无灯光**:视觉价值 = 翼面张合的**动态** + 造型本身

---

## 2. 定位与价值(收益封顶)

- **价值锚点**:视觉 / 动态炫酷("展翅感"),外行一眼能看懂
- **明确不为性能买单**:不追求下压力、不做气动调优、不测性能增益
- **场景**:100km/h 以内的日常骑行
- **ROI 闸门**:任何"为多一点性能 / 下压力"的复杂度,默认砍掉

---

## 3. 控制架构(防战线蔓延的生死线)

- **实时控制 100% 板载自包含**:MCU + 传感器,本地决定翼面开度
- **外部连接(蓝牙或有线)只做两件事**:① 配置(模式 / 参数) ② 数据采集与回传
- **外部端(手机 / PC / 网页)不参与实时控制**
- **无云、无 OTA、无跨学科集成**

---

## 4. 设计铁律(v1 四大死因的对策)

1. **收益封顶** — 价值是视觉,不是性能;砍掉一切性能向复杂度。
2. **第一周可观察纵切片** — 先做出"翼面随某个输入张合"的能看见的最小原型。
3. **单人可造、可调、可调试** — 不引入需专家 / 多人才能维护的结构。
4. **禁战线蔓延** — 无 App 实时控制、无云、无 OTA、无应变片性能反馈链、无多学科集成计划。
5. **外行可读** — 不懂的人看一眼就明白"它随速度展翅,很酷"。

---

## 5. 范围

### ✅ 做(In)
- **机械**:C 形翼面 + 作动机构 + 车头安装支架
- **电子**:板载 MCU + 测速 / 加减速传感 + 供电 + 作动驱动
- **软件**:板载实时控制逻辑 + 极简配置 / 数据接口(蓝牙或有线)

### ❌ 不做(Out — 明确放弃,守住边界)
- 主动下压力优化 / 闭环气动调参
- 配套 App 实时控制 / 云遥测 / OTA
- 应变片性能反馈链 / 五学科集成计划(v1 的坟墓)
- 灯光

---

## 6. 待裁决(Open Questions — 下一轮立项逐项拍板)

> 刻意保持精简(v1 曾有 28 项启动自检清单——那本身就是病征)。下面是真正卡住设计的少数硬议题。

| # | 议题 | 为什么是硬议题 |
|---|---|---|
| **OQ-1** | **作动方式**:纯被动气流压开(弹簧回位)/ 舵机闭环驱动 / 混合 | **头号**:直接决定全部机械与电子复杂度,是 v1 翻车的核心地带。被动 = 极简,舵机闭环 = 回到复杂老路 |
| OQ-2 | 安装平台:仍是 GSX250R?单车型还是通用支架 | 决定尺寸、安装方式、取电 |
| OQ-3 | 供电:取车电 vs 独立电池 | 影响布线复杂度与安装侵入性 |
| OQ-4 | 测速 / 状态来源:轮速 / GPS / 气流压差 / IMU | 决定传感成本与"加减速判定"如何实现 |
| OQ-5 | 开度-速度映射 + **安全限位** | 高速翼面过度张开 / 抖振 / 受横风 → 操控与安全风险,必须有上限与失效保护 |
| OQ-6 | 数据采集范围与用途 | 既然不做性能,采什么数据、给谁看、为什么采 |
| OQ-7 | 连接方式:蓝牙 / 有线 二选一还是都要 | 影响电路与配置端形态 |
| OQ-8 | 材料与制造:3D 打印 / 碳板 / 注塑 | 决定成本、重量、原型速度 |
| OQ-9 | 形态细节:C 形尺寸、单侧翼片数、自由度 | 决定"展翅"观感与机构复杂度 |
| OQ-10 | 安全 / 法规:车头突出物对行人、碰撞、视线、车检的影响 | 上路合规与安全底线 |
