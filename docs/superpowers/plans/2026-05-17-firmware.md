# Firmware Subsystem Plan — Starling Active Aero v1

> **Status:** Empty template. To be filled by `writing-plans`-dispatched subagent.

> **Source spec:** [`../specs/2026-05-17-active-front-aero-design.md`](../specs/2026-05-17-active-front-aero-design.md) §§ 5, 6

**Goal:** Deliver ESP-IDF C/C++ firmware for ESP32-S3 implementing 9-state priority-arbitrated control loop at 100 Hz, plus the `starlog_analyzer` Python package for offline data analysis. Firmware is bench-testable without physical wing/servo via sensor stubs.

**Bound by these Interface Contracts (from master plan):**
- **IC-1** — MCU pin map (every GPIO assignment + I2C address)
- **IC-2** — BLE GATT service / characteristic UUIDs + frame schemas
- **IC-3** — Binary log frame format (128 B packed, CRC16, little-endian)
- **IC-4** — Lookup table CSV format + piecewise linear interpolation rules
- **IC-5** — Firebase data model (for OTA firmware download + session metadata structure)
- **IC-6** — Power budget (must implement left/right servo 50 ms stagger to stay under PD limit)

**Tech stack:**
- ESP-IDF v5.x (Espressif official)
- C / C++17 (no exceptions, no STL containers in hot path)
- FreeRTOS (built into ESP-IDF)
- BLE: ESP-IDF NimBLE host
- WiFi STA: ESP-IDF WiFi
- OTA: ESP-IDF advanced_ota + dual-bank partition table
- Unit tests: Unity (ESP-IDF built-in)
- Python analyzer: numpy + pandas + matplotlib + pytest

**Files this plan will create or modify:**
- `firmware/CMakeLists.txt` + `sdkconfig` (ESP-IDF project)
- `firmware/main/main.c` — entry point + task scheduler
- `firmware/components/hal/` — hardware abstraction (sensors, servos, GPIO)
- `firmware/components/sensors/` — drivers (gps_neo_m9n.c, imu_bmi270.c, as5600.c, hall.c)
- `firmware/components/fusion/` — sensor fusion + Mahony filter
- `firmware/components/control/` — FSM + lookup table parser + control loop
- `firmware/components/safety/` — health checks + watchdog + fault state
- `firmware/components/logging/` — binary frame writer + SD card driver + ring buffer
- `firmware/components/comm/` — BLE GATT server + WiFi STA + HTTP server for log dl
- `firmware/components/ota/` — OTA client polling Firebase storage
- `firmware/attiny/attiny_watchdog.c` — separate 50-line ATtiny85 firmware
- `firmware/tests/` — Unity unit tests
- `analyzer/setup.py` + `analyzer/starlog_analyzer/` — Python package
- `analyzer/tests/` — pytest tests
- `analyzer/notebooks/` — example Jupyter notebooks

**Scope (~50 tasks expected):**

### Firmware Core
1. ESP-IDF project init + partition table (OTA dual-bank + storage)
2. HAL: GPIO + I2C + UART + PWM + ADC abstractions per IC-1 pin map
3. Driver: u-blox NEO-M9N UART parser (NMEA + UBX-NAV-PVT for 10 Hz binary)
4. Driver: BMI270 I2C + soft-reset + accel/gyro range + sample rate config
5. Driver: AS5600 ×2 with two I2C addresses (master IC-1)
6. Driver: A3144 Hall × 2 with interrupt + debounce
7. Driver: Servo PWM control (50 Hz, 1000-2000 µs pulse) with rate limiter
8. Driver: SD card SPI + FAT32 mount
9. Driver: ATtiny85 heartbeat output (50 ms toggle GPIO)
10. Fusion: 100 Hz speed fusion (GPS primary + Hall fallback)
11. Fusion: Mahony 50 Hz filter for roll/pitch (with vibration low-pass)
12. Fusion: Brake intent detection (Hall primary + IMU 0.3 g secondary)
13. Control: Lookup table parser (CSV reader + interpolator per IC-4)
14. Control: FSM with 9-state priority arbitration (per spec §5.2)
15. Control: State transition with hysteresis + rate-limited slew
16. Control: 100 Hz timer-driven control loop (run on dedicated FreeRTOS task)
17. Safety: 7-item health check (per spec §5.5) — each as separate function with severity
18. Safety: FAULT state handler (forces 0° + sets fault_flag)
19. Safety: Watchdog task (feed ESP32 internal WDT)
20. Logging: 128 B binary frame schema (per IC-3) + CRC16 CCITT
21. Logging: RAM ring buffer (1024 frames, ~130 KB)
22. Logging: 1 Hz flush task → SD card with file rotation (10 min / 100 MB)
23. Logging: File header 4 KB with session metadata
24. Comm: BLE GATT server (per IC-2 service + characteristics)
25. Comm: Telemetry notify @ 10 Hz (50 B binary frame)
26. Comm: Command write handler (JSON parser, dispatch to control)
27. Comm: Table read/write via BLE (gzip + chunk transfer)
28. Comm: WiFi STA mode (NVS-stored creds + STA reconnect)
29. Comm: HTTP server on WiFi for log file download (range requests)
30. OTA: ESP-IDF advanced_ota client polling Firebase Storage
31. OTA: Signature verification (use ESP-IDF secure_boot signed app)
32. OTA: Dual-bank rollback + 3-fail-disable
33. Calibration: zero-angle calibration command + persistent NVS storage
34. Calibration: IMU bias estimation (静置 5 分钟)
35. ATtiny85 firmware: 50-line C, monitor heartbeat, kill MOSFET on timeout
