# App Subsystem Plan — Starling Active Aero v1

> **Status:** Empty template. To be filled by `writing-plans`-dispatched subagent.
> **Master plan:** [`2026-05-17-active-aero-v1-master.md`](2026-05-17-active-aero-v1-master.md)
> **Source spec:** [`../specs/2026-05-17-active-front-aero-design.md`](../specs/2026-05-17-active-front-aero-design.md) §§ 6.4, 6.5

**Goal:** Deliver Flutter Android app (iOS scaffolded for v2) supporting BLE telemetry + WiFi STA file download + Firebase backend integration. APK is debug+release signed and installable on user's personal Android device for self-beta.

**Bound by these Interface Contracts (from master plan):**
- **IC-2** — BLE GATT service + characteristic UUIDs + JSON command schema
- **IC-3** — Binary log frame format (for parsing downloaded files)
- **IC-4** — Lookup table CSV format + piecewise linear semantics (for editor + simulation)
- **IC-5** — Firebase data model (Firestore collections + Storage paths)

**Tech stack:**
- Flutter 3.x + Dart 3.x
- State management: Riverpod
- BLE client: flutter_blue_plus (best Android BLE support)
- Charts: fl_chart (real-time + offline)
- Firebase: firebase_core, firebase_auth, cloud_firestore, firebase_storage, firebase_crashlytics, firebase_messaging
- Local storage: sqflite for session metadata + Hive for app preferences
- File handling: path_provider + archive (gzip)
- HTTP for WiFi STA: dio
- Test: flutter_test + mocktail
- CI: GitHub Actions for APK build

**Files this plan will create or modify:**
- `app/pubspec.yaml` + `app/analysis_options.yaml`

**Scope (~35 tasks expected):**

### Setup
1. Flutter project init + Riverpod + linter
2. Theme + color tokens + typography
3. Routing (go_router) with 7 pages
4. Firebase init (firebase_core, plumbing for android/app/google-services.json)

### Models + Services
5. Session / Bike / LookupTable / TelemetryFrame Dart models per IC-3, IC-4, IC-5
6. BLE client wrapper: scan + connect + characteristic notify per IC-2
7. BLE telemetry parser (50 B binary frame → TelemetryFrame model)
8. BLE command writer (JSON command + ack handling)
9. WiFi STA HTTP client for log file download
10. Starlog binary parser (.starlog → list of frames) per IC-3
11. Firebase auth wrapper (Google sign-in)
12. Firestore repo (sessions CRUD per IC-5 schema)
13. Storage repo (upload session.gz + download firmware)
14. Local sqflite cache for offline session metadata

### Pages
15. Dashboard page: real-time gauges + 30 s scrolling charts (fl_chart 10 Hz)
16. Recording page: start/stop + event mark + storage indicator
17. Sessions page: list + detail view + analysis charts
18. Sessions: braking event detector visualization
19. Table editor page: 9-state CSV editor + validation
20. Table editor: sandbox simulation (replay session with new table, show diff)
21. Calibration page: IMU bias / zero-angle wizard
22. Health page: live health bits + fault history + diagnostic dump
23. Settings page: WiFi creds + BLE pairing + bike profile + firmware version

### Backend Integration
24. Auto-upload session.gz on WiFi + charging
25. Firmware OTA notify (FCM push when new version available)
26. Cross-device sync (Firestore listener for session list)

### Tests
27. Unit test: BLE telemetry frame parser (round-trip)
28. Unit test: starlog binary parser (against fixture .starlog)
29. Unit test: lookup table CSV parser + validator
30. Widget test: Dashboard renders telemetry
31. Widget test: Table editor sandbox + revert

### Build + Release
32. Android release signing config + keystore (not committed)
33. APK build script
34. GitHub Actions CI building APK on push
35. README with install instructions for personal Android device
