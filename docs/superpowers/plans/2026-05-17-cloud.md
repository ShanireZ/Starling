# Cloud Subsystem Plan — Starling Active Aero v1

> **Status:** Expanded 2026-05-28; awaiting user review.
> **Master plan:** [`2026-05-17-active-aero-v1-master.md`](2026-05-17-active-aero-v1-master.md)
> **Source spec:** [`../specs/2026-05-17-active-front-aero-design.md`](../specs/2026-05-17-active-front-aero-design.md) §§ 6.6, 6.7, 6.8 (plus §§ 6.4–6.5 for app↔cloud handshake context)
> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use bite-sized TDD ordering (failing test → minimal impl → pass → commit).

**Goal:** Deliver Firebase project (Auth + Firestore + Storage + Cloud Functions) + minimal Web console (Next.js 14 App Router) for cross-session multi-bike analysis. Provides session upload pipeline (`.starlog.gz` → metrics → Firestore), OTA firmware distribution (release → FCM fanout), and a public project landing page. Designed to stay inside the Firebase Blaze free tier (≤ $10/month soft budget) for v1.

**Architecture:** A Firebase project in region **`asia-east2` (Hong Kong)** for China-friendly latency. Cloud Functions written in **Python 3.11** (same language as the firmware-plan `starlog_analyzer` so the frame decoder is a direct port — no re-implementation drift). Web Console is a **Next.js 14 App Router** site hosted on Firebase Hosting, served at the Firebase-default domain (custom domain deferred to v2). All schemas — collections, fields, storage paths, security-rule predicates — are **owned by IC-5** and cited at every touch point in this plan. The Firebase Local Emulator Suite (auth + firestore + storage + functions + hosting) is the **only** target for local dev and CI; live Firebase is touched only at deploy time.

**Tech stack:**
- **Firebase services:** Auth, Firestore (Native mode), Cloud Storage, Cloud Functions (2nd gen), Firebase Hosting, Firebase Cloud Messaging (FCM)
- **Billing plan:** Blaze (pay-as-you-go) — required for Cloud Functions outbound; v1 expected to stay inside free-tier quotas
- **Cloud Functions runtime:** Python 3.11 via the `firebase-functions` SDK v0.4+ (2nd-gen decorators `@storage_fn.on_object_finalized`, `@firestore_fn.on_document_created`, `@https_fn.on_call`)
- **Function deps:** `firebase-functions`, `firebase-admin`, `google-cloud-storage`, `google-cloud-firestore`, `numpy`, `pandas` (pinned in `requirements.txt`)
- **Test runner (functions):** pytest + `firebase-tools` emulator-suite + `requests` for HTTP probes
- **Web Console:** Next.js 14 (App Router) + TypeScript + Tailwind CSS + `react-firebase-hooks` (auth + firestore + storage) + `recharts` (charts) + `firebase` JS SDK v10
- **Web test runner:** Vitest + @testing-library/react for unit + widget; Playwright for one smoke e2e
- **CI/CD:** GitHub Actions on `main` push (two workflows: `deploy-functions.yml`, `deploy-web.yml`) using a service-account JSON in repo secret `FIREBASE_SA_KEY`
- **Tooling pinned versions:** `firebase-tools@13.x`, `node@20.x` (web build + emulator), `python@3.11.x` (functions venv)

## Bound Interface Contracts (verbatim — every byte/UUID/field below is load-bearing)

These are reproduced from the master plan and are **non-negotiable**. A wrong byte offset costs reprocessing every session in storage; a wrong Firestore field name breaks the App's queries silently. Every task that touches a contract re-cites the clause.

### IC-2 — BLE GATT (cloud needs only the data formats, not the GATT plumbing)

Cloud reads the firmware version + table version from telemetry-side fields (echoed by the App when it writes a session metadata doc). Source of truth from master IC-2:

```
Telemetry frame (50 bytes binary, little-endian, summary):
  uint64_t time_us           (off 0,  8B)
  uint16_t speed_kmh_cx100   (off 8,  2B)
  ...
  uint8_t  state_current     (off 18, 1B)   // enum value, see state_dwell key set below
  uint8_t  fault_flag        (off 19, 1B)
  ...
  uint8_t  table_version     (off 36, 1B)
  uint16_t crc16             (off 48, 2B)   // last 2 bytes
```

The Cloud Function does **not** parse BLE telemetry frames — those are App-only. But the **state-name enum** (the values that fill `state_dwell_percent` in IC-5) is shared across firmware (IC-3), app (IC-2 `state_current`), and cloud (this plan). Canonical names:

```
PARKED, LOW_SPEED, CRUISE, DRAG_REDUCE, BRAKING_LIGHT,
BRAKING_HARD, TRAIL_BRAKE, CORNERING, WHEELIE_GUARD, FAULT
```

(Per master plan OQ-FSM — to be frozen at Gate A. Cloud parser exports the same 10 names so Firestore queries align across subsystems.)

### IC-3 — Binary log frame (cloud parses `.starlog.gz` on Storage finalize)

Per master IC-3 (OQ-3 resolved 2026-05-24): **128 bytes/frame, packed, little-endian**; CRC16 CCITT (init `0xFFFF`, poly `0x1021`) in the **last 2 bytes** (offset 126); `lookup_table_version` is **uint8 at byte 99**. A **4 KB file header** (per spec §6.3 + firmware plan Task 22) precedes the frame stream with magic `0x53544152` ('STAR'), schema version, firmware version, bike profile, table hash/version, start time, rider note.

```
file layout = [4096-byte header] [128-byte frame] [128-byte frame] ...

frame layout (key offsets the cloud parser actually reads):
  off 0   uint64_t  time_us
  off 8   uint32_t  gps_time_unix
  off 12  int32_t   gps_lat_e7
  off 16  int32_t   gps_lon_e7
  off 22  uint16_t  gps_speed_kmh_cx100
  off 24  uint8_t   gps_fix_type
  off 56  int16_t   roll_cdeg
  off 58  int16_t   pitch_cdeg
  off 62  int16_t   linear_accel_long_cms2
  off 64  uint16_t  wheel_speed_kmh_cx100
  off 67  uint8_t   state_current
  off 88  uint8_t   fault_flag
  off 99  uint8_t   lookup_table_version   ← parser selector
  off 126 uint16_t  crc16
```

The cloud parser MUST be **byte-for-byte identical** to the firmware-plan analyzer (`analyzer/starlog_analyzer/frame.py`, firmware plan Tasks 43–44). Strategy: this plan **copies the analyzer module verbatim** into `cloud/functions/parsers/starlog.py` at first deploy and a golden-fixture round-trip test (Task 13) keeps them locked.

### IC-5 — Firebase data model (THIS PLAN OWNS IC-5)

The full IC-5 from master plan:

```
Firestore collections:
  users/{user_id}
    email, created_at, bikes: [bike_id]

  bikes/{bike_id}
    user_id, profile: "gsx250r" | "rc450", vin?: string,
    created_at, last_session_id

  sessions/{session_id}
    user_id, bike_id, start_time, duration_sec,
    distance_km, max_speed_kmh, fault_count,
    state_dwell_percent: {CRUISE: 75, BRAKING_LIGHT: 12, ...},
    storage_uri: "gs://bucket/sessions/<session_id>.starlog",
    firmware_version, table_version,
    analyzed_at?: timestamp,
    summary?: {braking_events: [...], peak_decel: ...}

  firmware_releases/{version}
    version: "0.1.0",
    storage_uri: "gs://bucket/firmware/0.1.0.bin",
    rollout_percentage: 100,
    released_at,
    changelog

Cloud Storage paths:
  /sessions/<session_id>.starlog (gzip-compressed binary log)
  /firmware/<version>.bin
  /lookup_tables/<bike_profile>/<version>.csv

Cloud Functions:
  - on_session_upload(): 解析 starlog → 提取关键指标 → 写 Firestore
  - on_firmware_publish(): 校验签名 → 通知所有用户
  - get_signed_url(): 生成 session 文件下载临时链接

Auth roles:
  - auth.uid == doc.user_id reads its own session
  - firmware_releases publicly readable
  - sessions writes only via Cloud Function (no direct client write)
```

> **IC-5 freeze rule:** If any task in this plan needs a field, collection, path, or rule predicate not listed above, **STOP**. Add an `> NOTE: requires master-plan amendment (IC-5)` callout at the relevant task and proceed only after the master plan is revised. Do not silently expand the schema.

### IC-6 — Power budget (cloud is not power-constrained — but FCM payload size matters)

Cloud has no power budget. The only consequence in this plan is that **OTA download is initiated by the firmware**, not pushed by Cloud — Cloud sends a small FCM data message ("new version available, fetch firmware_releases/<v>") and the firmware decides when to download. This keeps the firmware in control of its USB-PD 45 W ceiling.

## Task 4: Cloud Storage bucket structure (`/sessions`, `/firmware`, `/lookup_tables`)

**Files:**

- Modify: `cloud/README.md`

**Cost driver:** Storage free tier = 5 GB stored + 1 GB/day download. A `.starlog.gz` session is ~5 MB compressed (45 MB/hour raw, ~10× gzip on packed binary → ~4.5 MB/hour). One user, 10 sessions/month ≈ 50 MB → well inside free tier.

**Step 1: Create folder placeholders in default bucket**

Cloud Storage has no folders, only object paths. To match IC-5's `/sessions/<session_id>.starlog`, `/firmware/<version>.bin`, `/lookup_tables/<bike_profile>/<version>.csv` paths, we drop one zero-byte sentinel under each prefix so the console shows the structure.

`cloud/scripts/bootstrap_storage.sh`:

```bash
#!/usr/bin/env bash
set -euo pipefail
PROJECT_ID="${1:-starling-aero-v1}"
BUCKET="${PROJECT_ID}.appspot.com"

# IC-5 paths
for PREFIX in sessions firmware lookup_tables/gsx250r lookup_tables/rc450 lookup_tables/universal; do
  echo "" | gsutil cp - "gs://${BUCKET}/${PREFIX}/.keep"
done
echo "Bucket structure created: gs://${BUCKET}/{sessions,firmware,lookup_tables/<profile>}/"
```

**Step 2: Write the rules**

`cloud/firestore.rules`:

```javascript
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {

    // users/{uid}: only the user themselves
    match /users/{uid} {
      allow read, write: if request.auth != null && request.auth.uid == uid;
    }

    // bikes: owned by a user (IC-5: bikes.user_id)
    match /bikes/{bikeId} {
      allow read: if request.auth != null && request.auth.uid == resource.data.user_id;

"""CRC16 CCITT (init 0xFFFF, poly 0x1021). Mirrors IC-3 + firmware crc16.c byte-for-byte."""

def crc16_ccitt(data: bytes) -> int:
    crc = 0xFFFF
    for b in data:
        crc ^= b << 8
        for _ in range(8):
            crc = ((crc << 1) ^ 0x1021) & 0xFFFF if (crc & 0x8000) else (crc << 1) & 0xFFFF
    return crc
```

```python
"""Starlog binary log decoder — mirrors firmware analyzer/starlog_analyzer/frame.py (Task 44).

IC-3 byte-99 invariant: `lookup_table_version` is uint8 at byte 99 of a 128-byte
packed little-endian frame. CRC16 CCITT (init 0xFFFF, poly 0x1021) in the last 2 bytes.
4 KB file header (spec §6.3) precedes the frame stream.
"""
import struct
from .crc16 import crc16_ccitt

FRAME_SIZE = 128
TABLE_VERSION_OFFSET = 99      # IC-3 OQ-3 RESOLVED 2026-05-24
CRC_OFFSET = 126

FILE_HEADER_SIZE = 4096
FILE_MAGIC = 0x53544152        # 'STAR' (spec §6.3 + firmware Task 22)

# struct format string mirrors the firmware packed layout EXACTLY. Pads keep
# lookup_table_version at byte 99 and crc16 at byte 126.
_FRAME_FMT = (
    "<"                    # little-endian, packed
    "Q"                    # off 0   time_us
    "I"                    # off 8   gps_time_unix
    "i"                    # off 12  gps_lat_e7
    "i"                    # off 16  gps_lon_e7
    "h"                    # off 20  gps_alt_dm
    "H"                    # off 22  gps_speed_kmh_cx100
    "B"                    # off 24  gps_fix_type
    "B"                    # off 25  gps_hdop
    "B"                    # off 26  gps_sats
    "5x"                   # off 27..31  pad1
    "6h"                   # off 32..43  ax,ay,az,gx,gy,gz
    "12x"                  # off 44..55  pad2
    "h"                    # off 56  roll_cdeg
    "h"                    # off 58  pitch_cdeg
    "h"                    # off 60  yaw_cdeg
    "h"                    # off 62  linear_accel_long_cms2
    "H"                    # off 64  wheel_speed_kmh_cx100
    "B"                    # off 66  brake_lever_active
    "B"                    # off 67  state_current
    "h"                    # off 68  target_angle_l_cdeg
    "h"                    # off 70  target_angle_r_cdeg
    "h"                    # off 72  measured_angle_l_cdeg
    "h"                    # off 74  measured_angle_r_cdeg
    "H"                    # off 76  servo_current_l_ma
    "H"                    # off 78  servo_current_r_ma
    "8x"                   # off 80..87  pad3
    "B"                    # off 88  fault_flag
    "B"                    # off 89  health_bits
    "H"                    # off 90  supply_voltage_mv
    "H"                    # off 92  supercap_voltage_mv
    "H"                    # off 94  cpu_loop_us
    "3x"                   # off 96..98  pad_misc (aligns version to byte 99)
    "B"                    # off 99  lookup_table_version  ← IC-3 invariant
    "26x"                  # off 100..125  pad4
    "H"                    # off 126 crc16
)
assert struct.calcsize(_FRAME_FMT) == FRAME_SIZE, "frame layout != 128 B"

_FRAME_FIELDS = (
    "time_us","gps_time_unix","gps_lat_e7","gps_lon_e7","gps_alt_dm",
    "gps_speed_kmh_cx100","gps_fix_type","gps_hdop","gps_sats",
    "ax","ay","az","gx","gy","gz",
    "roll_cdeg","pitch_cdeg","yaw_cdeg","linear_accel_long_cms2",
    "wheel_speed_kmh_cx100","brake_lever_active","state_current",
    "target_angle_l_cdeg","target_angle_r_cdeg","measured_angle_l_cdeg","measured_angle_r_cdeg",
    "servo_current_l_ma","servo_current_r_ma",
    "fault_flag","health_bits","supply_voltage_mv","supercap_voltage_mv","cpu_loop_us",
    "lookup_table_version",
    "crc16",
)

def decode_frame(raw: bytes) -> dict:
    if len(raw) != FRAME_SIZE:
        raise ValueError(f"frame must be {FRAME_SIZE} bytes, got {len(raw)}")
    values = struct.unpack(_FRAME_FMT, raw)
    out = dict(zip(_FRAME_FIELDS, values))
    out["crc_ok"] = (crc16_ccitt(raw[:CRC_OFFSET]) == out["crc16"])
    return out

# File header — spec §6.3 + firmware Task 22.
# Layout: uint32 magic, uint16 schema_version, char[16] firmware_version,
#         char[16] bike_profile, uint32 table_version, uint64 start_time_us,
#         char[128] rider_note, [reserved padding to 4096].
_HEADER_FMT = "<I H 16s 16s I Q 128s"
_HEADER_FIXED_SIZE = struct.calcsize(_HEADER_FMT)
assert _HEADER_FIXED_SIZE <= FILE_HEADER_SIZE

def decode_file_header(raw: bytes) -> dict:
    if len(raw) < _HEADER_FIXED_SIZE:
        raise ValueError(f"header too short: {len(raw)} < {_HEADER_FIXED_SIZE}")
    magic, schema_ver, fw, profile, tbl_ver, start_us, note = struct.unpack(
        _HEADER_FMT, raw[:_HEADER_FIXED_SIZE]
    )
    if magic != FILE_MAGIC:
        raise ValueError(f"bad magic {magic:#x}, expected {FILE_MAGIC:#x}")
    return {
        "magic": magic,
        "schema_version": schema_ver,
        "firmware_version": fw.rstrip(b"\x00").decode("utf-8"),
        "bike_profile":    profile.rstrip(b"\x00").decode("utf-8"),
        "table_version":   tbl_ver,
        "start_time_us":   start_us,
        "rider_note":      note.rstrip(b"\x00").decode("utf-8", errors="replace"),
    }
```

"""Session metrics → IC-5 sessions doc fields.

Output fields match IC-5 sessions schema exactly. State enum names must match
firmware/IC-2 (PARKED..FAULT, 10 entries — finalized at Gate A per master OQ-FSM).
"""
import math
import pandas as pd

CANONICAL_STATES = (
    "PARKED", "LOW_SPEED", "CRUISE", "DRAG_REDUCE", "BRAKING_LIGHT",
    "BRAKING_HARD", "TRAIL_BRAKE", "CORNERING", "WHEELIE_GUARD", "FAULT",
)

def state_dwell_percent(df: pd.DataFrame, state_names=CANONICAL_STATES) -> dict:
    """Return {<state name>: percent} summing to ~100. Zero-pads missing states
    so the IC-5 doc shape is stable across sessions."""
    out = {n: 0 for n in state_names}
    if len(df) == 0:
        return out
    counts = df["state_current"].value_counts().to_dict()
    total = len(df)
    for enum_value, n in counts.items():
        if 0 <= int(enum_value) < len(state_names):
            out[state_names[int(enum_value)]] = round(100 * n / total)
    # Round-up the largest bucket to make sum == 100 exactly
    drift = 100 - sum(out.values())
    if drift != 0:
        biggest = max(out, key=out.get)
        out[biggest] += drift
    return out

def peak_decel_g(df: pd.DataFrame) -> float:
    """Largest deceleration magnitude in g, from linear_accel_long_cms2 (cm/s²)."""
    if "linear_accel_long_cms2" not in df or len(df) == 0:
        return 0.0
    most_neg_cms2 = df["linear_accel_long_cms2"].min()  # most negative = deceleration
    return abs(most_neg_cms2) / 100.0 / 9.80665

def detect_braking_events(df, min_decel_g=0.3, sample_hz=100) -> list:
    """Group contiguous frames with decel >= threshold. Returns list of dicts:
       {entry_speed_kmh, exit_speed_kmh, peak_decel_g, duration_sec, time_us_start}."""
    if len(df) == 0:
        return []
    decel_g = -df["linear_accel_long_cms2"] / 100.0 / 9.80665
    above = decel_g >= min_decel_g
    events = []
    start = None
    for i in range(len(df)):
        if bool(above.iloc[i]) and start is None:
            start = i
        elif not bool(above.iloc[i]) and start is not None:
            events.append((start, i - 1)); start = None
    if start is not None:
        events.append((start, len(df) - 1))
    out = []
    for a, b in events:
        seg = df.iloc[a:b + 1]
        seg_decel = decel_g.iloc[a:b + 1]
        out.append({
            "entry_speed_kmh": float(seg["gps_speed_kmh_cx100"].iloc[0]) / 100.0,
            "exit_speed_kmh":  float(seg["gps_speed_kmh_cx100"].iloc[-1]) / 100.0,
            "peak_decel_g":    float(seg_decel.max()),
            "duration_sec":    (b - a + 1) / float(sample_hz),
            "time_us_start":   int(seg["time_us"].iloc[0]) if "time_us" in seg else 0,
        })
    return out

def summarize_session(df: pd.DataFrame, header: dict) -> dict:
    """Build the IC-5 sessions doc payload (minus session_id, user_id, bike_id, storage_uri —
    those are added by the on_session_upload handler)."""
    if len(df) == 0:
        return {
            "duration_sec": 0, "distance_km": 0.0,
            "max_speed_kmh": 0.0, "fault_count": 0,
            "state_dwell_percent": {n: 0 for n in CANONICAL_STATES},
            "firmware_version": header.get("firmware_version", "0.0.0"),
            "table_version":    header.get("table_version", 0),
            "summary": {"braking_events": [], "peak_decel": 0.0},
        }
    # duration: last-first time_us, fall back to row count / sample_hz
    if "time_us" in df:
        duration_sec = int((df["time_us"].iloc[-1] - df["time_us"].iloc[0]) / 1_000_000)
    else:
        duration_sec = len(df) // 100
    # distance: integrate gps speed over duration (very rough; firmware-side analyzer
    # offers a better number — this is just a session header)
    speed_kmh = df["gps_speed_kmh_cx100"] / 100.0
    distance_km = float((speed_kmh.mean() * (duration_sec / 3600.0)))
    return {
        "duration_sec":  duration_sec,
        "distance_km":   round(distance_km, 3),
        "max_speed_kmh": round(float(speed_kmh.max()), 1),
        "fault_count":   int((df["fault_flag"] > 0).sum()) if "fault_flag" in df else 0,
        "state_dwell_percent": state_dwell_percent(df),
        "firmware_version":    header.get("firmware_version", "0.0.0"),
        "table_version":       int(header.get("table_version", 0)),
        "summary": {
            "braking_events": detect_braking_events(df, min_decel_g=0.3),
            "peak_decel":     peak_decel_g(df),
        },
    }
```

"""on_session_upload — triggered by Storage finalize on sessions/*.

Per IC-5: extract metrics → write Firestore sessions/{session_id}.
The Admin SDK bypasses the 'sessions writes only via Cloud Function' rule (Task 5).
"""
import gzip, io, os, re, logging
from datetime import datetime, timezone
from firebase_admin import initialize_app, firestore, storage
from firebase_functions import storage_fn, options

from parsers.starlog import decode_file_header, decode_frame, FRAME_SIZE, FILE_HEADER_SIZE
from metrics.extractor import summarize_session
import pandas as pd

# Lazy init — Functions cold-start otherwise hits initialize_app twice.
_app = None
def _ensure_app():
    global _app
    if _app is None:
        _app = initialize_app()
    return _app

log = logging.getLogger("session_upload")

# IC-5 path: sessions/<user_uid>_<datetime>.starlog  (uploaded by App per Task 6 rules)
# We rename to canonical sessions/<session_id>.starlog while writing the Firestore doc.
SESSIONS_PREFIX = "sessions/"
UID_NAME_RE = re.compile(r"^sessions/(?P<uid>[A-Za-z0-9]+)_(?P<rest>.+)\.starlog(?:\.gz)?$")


@storage_fn.on_object_finalized(region="asia-east2", memory=options.MemoryOption.MB_512)
def on_session_upload(event: storage_fn.CloudEvent[storage_fn.StorageObjectData]) -> None:
    _ensure_app()
    obj = event.data
    name = obj.name or ""
    if not name.startswith(SESSIONS_PREFIX):
        log.info("ignore non-session upload: %s", name); return

    m = UID_NAME_RE.match(name)
    if not m:
        log.warning("session upload doesn't match <uid>_<rest>.starlog pattern: %s", name)
        return
    user_uid = m.group("uid")

    # Download + (optionally) gunzip
    bucket = storage.bucket(obj.bucket)
    blob = bucket.blob(name)
    raw_compressed = blob.download_as_bytes()
    raw = gzip.decompress(raw_compressed) if name.endswith(".gz") else raw_compressed

    if len(raw) < FILE_HEADER_SIZE + FRAME_SIZE:
        log.error("starlog too small: %d bytes", len(raw)); return

    header = decode_file_header(raw[:FILE_HEADER_SIZE])
    rows = []
    crc_bad = 0
    for off in range(FILE_HEADER_SIZE, len(raw) - FRAME_SIZE + 1, FRAME_SIZE):
        fr = decode_frame(raw[off:off + FRAME_SIZE])
        if not fr["crc_ok"]:
            crc_bad += 1
            continue
        rows.append(fr)
    df = pd.DataFrame(rows)
    summary = summarize_session(df, header)

    # session_id: stable hash-free composition — bike profile + start_time_us
    session_id = f"{header['bike_profile']}_{header['start_time_us']}"

    # Resolve user's primary bike
    db = firestore.client()
    bike_id = _resolve_bike_id(db, user_uid, header["bike_profile"])

    # Move object to canonical path: sessions/<session_id>.starlog(.gz)
    canonical_name = f"sessions/{session_id}.starlog" + (".gz" if name.endswith(".gz") else "")
    if canonical_name != name:
        bucket.copy_blob(blob, bucket, new_name=canonical_name)
        blob.delete()
    storage_uri = f"gs://{obj.bucket}/{canonical_name}"

    doc = {
        "user_id":      user_uid,
        "bike_id":      bike_id,
        "start_time":   _us_to_datetime(header["start_time_us"]),
        "storage_uri":  storage_uri,
        "analyzed_at":  datetime.now(timezone.utc),
        "crc_bad_frames": crc_bad,
        **summary,
    }
    db.collection("sessions").document(session_id).set(doc)
    log.info("session_upload: wrote sessions/%s (%d frames, %d crc-bad)",
             session_id, len(df), crc_bad)


def _resolve_bike_id(db, user_uid: str, bike_profile: str) -> str:
    """Look up the first matching bike for this user, else return 'unknown'.
    IC-5: bikes.user_id + bikes.profile."""
    q = (db.collection("bikes")
           .where("user_id", "==", user_uid)
           .where("profile", "==", bike_profile)
           .limit(1).stream())
    for d in q:
        return d.id
    return "unknown"


def _us_to_datetime(time_us: int) -> datetime:
    return datetime.fromtimestamp(time_us / 1_000_000.0, tz=timezone.utc)
```

"""on_firmware_publish — triggered by Firestore document create on firmware_releases/{version}.

Per IC-5: sends FCM data message to all users whose bikes match `bike_profile`.
Verifies the firmware blob exists in Storage before fanning out.
Per IC-6: data-only (silent) message — firmware decides when to download.
"""
import logging
from firebase_admin import firestore, storage, messaging
from firebase_functions import firestore_fn, options
from .session_upload import _ensure_app

log = logging.getLogger("firmware_publish")


def build_fcm_message(version: str, bike_profile: str, changelog: str) -> messaging.Message:
    return messaging.Message(
        data={
            "op": "firmware_available",
            "version": version,
            "bike_profile": bike_profile,
            "changelog": (changelog or "")[:1024],   # FCM data field cap is 4 KB total
        },
        topic=None,  # caller sets per-user topic
    )


def eligible_user_topics(bikes: list[dict], bike_profile: str) -> list[str]:
    """Return unique FCM topic strings for users whose bike profile matches.
    Topic convention: 'bike-<profile>-<user_id>'."""
    seen = set()
    out = []
    for b in bikes:
        if b.get("profile") == bike_profile:
            t = f"bike-{bike_profile}-{b['user_id']}"
            if t not in seen:
                seen.add(t); out.append(t)
    return out


@firestore_fn.on_document_created(
    document="firmware_releases/{version}", region="asia-east2",
    memory=options.MemoryOption.MB_256,
)
def on_firmware_publish(event: firestore_fn.Event[firestore_fn.DocumentSnapshot | None]) -> None:
    _ensure_app()
    snap = event.data
    if snap is None:
        return
    rel = snap.to_dict() or {}
    version = rel.get("version") or event.params["version"]
    storage_uri = rel.get("storage_uri", "")
    rollout = int(rel.get("rollout_percentage", 0))
    changelog = rel.get("changelog", "")
    bike_profile = rel.get("bike_profile", "universal")

    if rollout <= 0:
        log.info("firmware %s rollout_percentage=0 — no fanout", version); return
    # Verify the .bin actually exists in Storage at the recorded URI
    if storage_uri.startswith("gs://"):
        bucket_name, _, object_name = storage_uri[5:].partition("/")
        if not storage.bucket(bucket_name).blob(object_name).exists():
            log.error("firmware %s: storage_uri %s does not exist — abort fanout",
                      version, storage_uri); return

    # Resolve eligible users
    db = firestore.client()
    bikes = [b.to_dict() | {"_id": b.id} for b in db.collection("bikes")
             .where("profile", "==", bike_profile).stream()]
    topics = eligible_user_topics(bikes, bike_profile)

    # Random subset by rollout %
    if rollout < 100:
        import hashlib
        topics = [t for t in topics
                  if (int(hashlib.sha256(f"{version}:{t}".encode()).hexdigest(), 16) % 100) < rollout]

    sent = 0
    for t in topics:
        m = build_fcm_message(version, bike_profile, changelog)
        m.topic = t
        try:
            messaging.send(m)
            sent += 1
        except Exception as e:
            log.warning("FCM send failed for topic %s: %s", t, e)
    log.info("firmware_publish: %s notified %d/%d topics (rollout %d%%)",
             version, sent, len(topics), rollout)
```

"""get_signed_url — HTTPS callable returning a short-lived signed URL for a session blob.

Per IC-5: only the owner (auth.uid == session.user_id) may fetch their own session URL.
"""
import datetime as dt
from firebase_admin import firestore, storage
from firebase_functions import https_fn, options
from .session_upload import _ensure_app


@https_fn.on_call(region="asia-east2", memory=options.MemoryOption.MB_256)
def get_signed_url(req: https_fn.CallableRequest) -> dict:
    _ensure_app()
    if req.auth is None:
        raise https_fn.HttpsError(https_fn.FunctionsErrorCode.UNAUTHENTICATED,
                                  "Must be signed in.")
    sid = (req.data or {}).get("session_id")
    if not sid:
        raise https_fn.HttpsError(https_fn.FunctionsErrorCode.INVALID_ARGUMENT,
                                  "session_id required")

    db = firestore.client()
    doc = db.collection("sessions").document(sid).get()
    if not doc.exists:
        raise https_fn.HttpsError(https_fn.FunctionsErrorCode.NOT_FOUND, "session not found")
    sd = doc.to_dict()
    if sd.get("user_id") != req.auth.uid:
        raise https_fn.HttpsError(https_fn.FunctionsErrorCode.PERMISSION_DENIED,
                                  "Not your session.")

    uri = sd["storage_uri"]                   # gs://bucket/sessions/<id>.starlog
    bucket_name, _, obj_name = uri[5:].partition("/")
    blob = storage.bucket(bucket_name).blob(obj_name)

    expires_in_sec = 15 * 60
    url = blob.generate_signed_url(
        version="v4",
        expiration=dt.timedelta(seconds=expires_in_sec),
        method="GET",
    )
    return {"url": url, "expires_at": int(dt.datetime.utcnow().timestamp()) + expires_in_sec}
```

             signInGoogle: () => Promise<void>; signOutNow: () => Promise<void>; };

                                     signInGoogle: async()=>{}, signOutNow: async()=>{} });

"use client";
import { useState } from "react";
import { ref, uploadBytes } from "firebase/storage";
import { doc, setDoc, serverTimestamp } from "firebase/firestore";
import { storage, db } from "../lib/firebase";
import RolloutSlider from "./RolloutSlider";

export default function FirmwareUploader() {
  const [version, setVersion] = useState("");
  const [bikeProfile, setBikeProfile] = useState<"gsx250r"|"rc450"|"universal">("universal");
  const [rollout, setRollout] = useState(10);
  const [changelog, setChangelog] = useState("");
  const [file, setFile] = useState<File | null>(null);
  const [status, setStatus] = useState<string>("");

  async function publish() {
    if (!file || !version) { setStatus("version + file required"); return; }
    if (!/^\d+\.\d+\.\d+$/.test(version)) { setStatus("version must be semver X.Y.Z"); return; }
    setStatus("uploading...");
    const path = `firmware/${version}.bin`;
    await uploadBytes(ref(storage, path), file);
    // Writing the Firestore doc triggers on_firmware_publish (Task 14)
    await setDoc(doc(db, "firmware_releases", version), {
      version,
      storage_uri: `gs://${process.env.NEXT_PUBLIC_FIREBASE_STORAGE_BUCKET}/${path}`,
      rollout_percentage: rollout,
      bike_profile: bikeProfile,   // ← IC-5 amendment candidate (Task 14 NOTE)
      released_at: serverTimestamp(),
      changelog: changelog.slice(0, 4000),
    });
    setStatus(`published ${version} → ${bikeProfile} @ ${rollout}%`);
  }

  return (
    <div className="space-y-4 max-w-xl">
      <label className="block">
        <span className="text-sm font-semibold">Version (semver)</span>
        <input value={version} onChange={(e) => setVersion(e.target.value)}
               placeholder="0.1.1" className="w-full border rounded px-3 py-2" />
      </label>
      <label className="block">
        <span className="text-sm font-semibold">Bike profile</span>
        <select value={bikeProfile} onChange={(e) => setBikeProfile(e.target.value as any)}
                className="w-full border rounded px-3 py-2">

      </label>
      <label className="block">

100% threshold, uncomment the `disable_billing()` call below AND grant the
service account the 'Billing Account Administrator' role.
For v1 we keep it log-only so accidental triggers don't brick the project.
"""
import base64, json, logging
from firebase_functions import pubsub_fn, options

log = logging.getLogger("budget_killswitch")

HARD_CAP_USD = 50.0   # matches Task 7 budget config


def parse_budget_event(payload: dict) -> dict:
    """Decode the Pub/Sub message body (base64-encoded JSON per GCP billing schema)."""
    raw = payload.get("data", "")
    if raw:
        return json.loads(base64.b64decode(raw).decode())
    return payload


def should_disable(ev: dict) -> bool:
    return float(ev.get("costAmount", 0)) >= float(ev.get("budgetAmount", HARD_CAP_USD))


@pubsub_fn.on_message_published(topic="budget-alerts", region="asia-east2",
                                 memory=options.MemoryOption.MB_256)
def on_budget_alert(event):
    body = json.loads(event.data.message.data.decode()) if event.data.message.data else {}
    log.warning("budget event: cost=%s budget=%s alert=%s",
                body.get("costAmount"), body.get("budgetAmount"), body.get("alertThresholdExceeded"))
    if should_disable(body):
