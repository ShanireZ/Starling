# Cloud Subsystem Plan — Starling Active Aero v1

**Goal:** Deliver Firebase project (Auth + Firestore + Storage + Cloud Functions) + minimal Web console (Next.js) for cross-session multi-bike analysis. Provides session upload pipeline + OTA firmware distribution + future-ready for community table sharing.

**Bound by these Interface Contracts (from master plan):**
- **IC-2** — Data formats (BLE frame for compatibility with app uploads)
- **IC-3** — Binary log frame format (Cloud Function parses `.starlog.gz`)
- **IC-5** — Firestore collections + Storage paths + Auth roles (this plan **owns** IC-5)

**Tech stack:**
- Firebase project (Spark or Blaze plan — Blaze required for Cloud Functions but free tier covers v1 usage)
- Firebase services: Auth, Firestore, Storage, Cloud Functions, Crashlytics, Cloud Messaging (FCM)
- Cloud Functions: Python 3.11 runtime (Node.js also acceptable — choose Python for consistency with analyzer)
- Web console: Next.js 14 (App Router) + Tailwind CSS + react-firebase-hooks
- Hosting: Firebase Hosting
- CI/CD: GitHub Actions deploying to Firebase
- Monitoring: Firebase Performance Monitoring + GCP Logging

**Files this plan will create or modify:**
- `cloud/firebase.json` — Firebase project config
- `cloud/firestore.rules` — security rules per IC-5
- `cloud/firestore.indexes.json` — composite indexes
- `cloud/storage.rules` — Storage security rules
- `cloud/functions/main.py` — Cloud Functions entry (Python)
- `cloud/functions/parsers/starlog.py` — `.starlog.gz` parser (port from analyzer)
- `cloud/functions/handlers/session_upload.py` — on_session_upload trigger
- `cloud/functions/handlers/firmware_publish.py` — on_firmware_publish trigger
- `cloud/functions/handlers/signed_url.py` — get_signed_url HTTPS
- `cloud/functions/requirements.txt`
- `cloud/web/` — Next.js app (pages: login, sessions, session detail, firmware mgmt)
- `cloud/web/app/` — Next.js App Router pages
- `cloud/web/components/` — shared UI
- `cloud/.github/workflows/deploy.yml` — CI/CD
- `cloud/README.md` — Setup + deploy + cost notes

**Scope (~25 tasks expected):**

### Firebase Setup
1. Firebase project create + region (asia-east2 Hong Kong for China-friendly latency)
2. Auth providers (Google + Email/Password) enable
3. Firestore database create + initial collections per IC-5
4. Storage buckets create + structure per IC-5 (`/sessions`, `/firmware`, `/lookup_tables`)
5. Security rules: Firestore (`auth.uid == doc.user_id` per IC-5)
6. Security rules: Storage (sessions private; firmware public-read; lookup_tables public-read)
7. Composite indexes (sessions: user_id + start_time DESC)

### Cloud Functions
8. Cloud Functions Python project setup + deploy
9. Function: `on_session_upload` — triggered by Storage finalize event
10. Function: parse `.starlog.gz` (port from `starlog_analyzer`)
11. Function: extract metrics → write Firestore session doc
12. Function: `on_firmware_publish` — trigger on `firmware_releases` doc create
13. Function: send FCM notification to all users with matching `bike_profile`
14. Function: `get_signed_url` — HTTPS endpoint generating temp download links
15. Local emulator suite testing

### Web Console (minimal)
16. Next.js project init + Tailwind + firebase-hooks
17. Login page (Google sign-in)
18. Sessions page: list user's sessions
19. Session detail: speed/state/wing-angle time series (recharts or fl_chart-like)
20. Firmware management page: upload new firmware + rollout %
21. Public Web docs page (project overview for casual visitors)

### CI/CD + Ops
22. GitHub Actions deploying functions on push to main
23. GitHub Actions deploying web console to Firebase Hosting
24. Cost monitoring dashboard + budget alert ($10/month threshold)
25. README + runbook (deploy / rollback / disaster recovery)
