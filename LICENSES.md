# Licensing

Starling uses a split license model because the repository contains open
hardware design files, software, and documentation.

| Path / material | License |
|---|---|
| Hardware design files, including `cad/`, `pcb/`, schematics, Gerbers, mechanical drawings, manufacturing exports, assembly files, and hardware BOMs | [CERN-OHL-S-2.0](LICENSE-CERN-OHL-S-2.0.txt) |
| Software, including `firmware/`, `app/`, `cloud/`, `analyzer/`, `scripts/`, and source code under `code/` | [Apache-2.0](LICENSE-Apache-2.0.txt) |
| Documentation, including `docs/`, `README.md`, `README.zh-CN.md`, specifications, plans, manuals, checklists, and test reports | [CC BY-SA 4.0](LICENSE-CC-BY-SA-4.0.txt) |

The Starling name, logo, and project branding are not licensed for trademark
use by these files unless explicitly stated elsewhere.

This licensing map is intended for repository organization. If a file contains
an SPDX identifier or a more specific license notice, that file-specific notice
takes precedence.
